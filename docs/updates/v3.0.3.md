### 系统优化

### 知识库相关

#### 新增功能：统一对话决策契约

- 一轮对话可以同时承载多个目标，并分别跟踪执行状态。
- 反问、选项确认和补充输入会继续处理原始目标，不再创建无关的新任务。
- 每次工具调用都会绑定对应目标，避免录入、查询、修改之间相互串线。
- 写入操作必须获得明确授权，查询结果不能被错误转化为录入操作。
- 每个目标都会明确结束为完成、取消或等待用户补充。

#### 新增功能：多文件与多来源处理

- 支持单轮对话同时处理多个附件和多个来源。
- 文件与输入框内容统一归入当前对话目标，不再互相覆盖。
- 多文件处理过程中保留每份文件的来源关系和处理目的。
- 附件相关目标经过反问、确认和继续执行后仍能保持完整。

#### 功能优化

- 优化非理想输入处理，增强对口语、省略表达、连续补充和模糊描述的理解能力。
- 优化知识目标排序：标题、章节路径和结构化意图优先于正文中的偶然关键词。
- 当只有一个明确目标时直接生成变更计划；真正存在多个合理目标时才触发反问。
- 反问候选增加章节名称、完整路径和现行内容预览，方便用户准确选择。
- 修改整段内容时可以识别真正需要变化的核心语句，保留其他无关事实。
- 重复录入仅存在空格、换行或标点差异时，识别为无需修改。
- 优化来源覆盖判断，减少重复召回和无意义的来源补全循环。
- 优化答案事实校验，JSON、模板、命令和文档示例不再被误判为无来源事实。
- 保留代码块和可复制内容的原始格式，不再破坏缩进与空白。

#### 缺陷修复

- 修复反问后丢失原始目标、上下文或附件范围的问题。
- 修复工具被内部重定向后，Trace 和实际执行工具不一致的问题。
- 修复 ReAct 达到步骤上限时目标被遗失或错误结束的问题。
- 修复模型不可用、用户取消或执行被拒绝后目标仍保持运行状态的问题。
- 修复移动原文后遗留旧反问，导致后续对话继续使用过期上下文的问题。
- 修复多来源附件只保留部分目标的问题。
- 修复错误来源覆盖判断导致处理反复循环的问题。
- 修复示例字段被当作业务事实，从而导致最终答案被拒绝的问题。
- 修复无实际变化的录入仍产生新版本的问题。
- 修复正文中出现相同关键词时产生过多错误候选的问题。
- 修复修改单条规则时意外删除同章节其他事实的问题。
- 修复文本修改后出现重复标点的问题。
- 修复并发召回时可能出现的模型加载冲突和 `Broken pipe` 异常。

### AsyncTest Mind MCP 相关

#### 新增功能：AsyncTest Mind MCP 操作控制
- AI 操作思维导图时，会显示“Agent 操作中...”。
- 用户可以随时退出控制，阻止 AI 继续修改。
- 重新恢复控制时，需要用户确认。

#### 功能优化
- AI 新增和修改节点时，画面会跟随正在操作的节点。
- 批量添加节点更加流畅，减少卡顿和突然刷新。
- 优化 MCP 返回内容，减少 AI 读取和分析时间。
- 批量修改支持正常撤销和重做。

#### 缺陷修复
- 修复批量新增节点后布局重叠的问题。
- 修复 MCP 操作后撤销结果不正确的问题。
- 修复退出控制后 AI 仍可能继续修改的问题。
- 修复保存内容与当前窗口内容不一致的问题。
